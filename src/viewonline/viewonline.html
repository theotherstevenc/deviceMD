Assets used in HTML (images, external stylesheets, etc) are referenced from a location, and the location can be identified using HTTP_REFERER.

However, if either the HTML or the assets used in the HTML is accessed via a local file URL, the HTTP_REFERER is not passed.

Since all email clients block the HTTP_REFERER, we can generate CSS for viewing hosted/online versions of our templates.

green background: external css is supported
red background: external css is not supported
